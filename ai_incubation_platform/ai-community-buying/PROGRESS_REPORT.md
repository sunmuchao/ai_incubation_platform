### 2. 核心业务规则实现
#### 库存管理规则
- 预锁定机制：参团时锁定库存，避免超卖
- 成团扣减：团购成功后正式扣减库存
- 失败回滚：团购失败/过期/取消时自动解锁库存
- 状态自动流转：库存为0自动售罄，补货自动恢复

#### 团购状态流转
```
OPEN → SUCCESS → 生成订单
     → EXPIRED → 解锁库存
     → FAILED → 解锁库存
     → CANCELLED → 解锁库存
```

## 二、技术架构亮点

### 1. 分层架构设计
```
API 层 → 服务层 → 模型层
```
- 接口与业务逻辑分离，便于后续扩展
- 核心业务逻辑高度内聚，易于测试和维护

### 2. 错误处理体系
- 自定义异常类型：库存不足、团购关闭、满员、重复加入等
- 统一的错误码和错误信息，便于前端处理
- 完整的日志记录，支持问题追踪

### 3. 可扩展性设计
- 枚举类型定义所有状态，避免硬编码
- 服务层依赖抽象，便于后续接入持久化存储
- 接口设计符合 RESTful 规范，便于前端接入
