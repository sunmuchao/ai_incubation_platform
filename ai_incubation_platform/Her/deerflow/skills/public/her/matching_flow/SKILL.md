# Her 智能匹配流程

这是一个完整的"从认识到约会"的智能流程 Skill。

## 使用场景

当用户表达以下意图时，应该使用此流程：

- "帮我找对象"
- "我想找个男朋友/女朋友"
- "帮我匹配一下"
- "有没有合适的人推荐"
- "今日推荐"

## 执行流程

此 Skill 描述了一个完整的匹配流程，Agent 应根据用户反馈动态调整：

### Phase 1: 找匹配对象

1. 调用 `her_find_matches` 获取候选对象
2. 展示匹配结果给用户：
   - 姓名和年龄
   - 匹配度
   - 共同兴趣
   - 推荐理由

### Phase 2: 询问意向

根据匹配结果，主动询问用户：

- "你对 [姓名] 感兴趣吗？我可以帮你分析匹配度"
- 如果用户说"是"或表达兴趣 → 进入 Phase 3
- 如果用户说"不喜欢" → 展示下一个候选对象
- 如果用户说"都要" → 为多个对象生成破冰建议

### Phase 3: 深度分析（可选）

如果用户对某个对象感兴趣：

1. 调用 `her_analyze_compatibility` 分析匹配度
2. 展示分析结果：
   - 总体匹配度
   - 各维度分析（兴趣、性格、生活方式、目标）
   - 注意事项和建议

### Phase 4: 破冰建议

1. 调用 `her_get_icebreaker` 生成开场白
2. 提供多个破冰建议供用户选择：
   - 不同的风格（友好、直接、幽默）
   - 基于 TA 的兴趣爱好定制

### Phase 5: 约会策划（可选）

如果用户想进一步发展：

1. 询问约会地点偏好
2. 调用 `her_plan_date` 策划约会方案
3. 提供完整的约会建议：
   - 活动类型
   - 地点推荐
   - 注意事项

## Agent 行为指南

1. **不要一次性调用所有工具** - 根据用户反馈动态调整
2. **主动询问** - 不要等用户说下一步
3. **个性化推荐** - 根据用户画像（年龄、城市、兴趣）定制
4. **保持对话流畅** - 不要每次都重复相同的话
5. **优雅降级** - 如果某个工具失败，继续流程其他部分
6. **记住用户回答** - 用户表达偏好时，立即调用 `her_update_preference` 写入数据库

## 偏好更新时机

当用户表达以下偏好时，**必须**调用 `her_update_preference`：
- 年龄范围："年龄相仿的"、"找20-30岁的"
- 地点偏好："只找无锡同城"、"接受异地"
- 关系目标："奔着结婚去的"、"认真谈恋爱"

示例：
```
用户: "年龄相仿的、只找无锡同城的、奔着结婚去的"

Agent 应调用:
- her_update_preference(dimension="preferred_age_min", value="24")
- her_update_preference(dimension="preferred_age_max", value="32")
- her_update_preference(dimension="preferred_location", value="无锡")
- her_update_preference(dimension="accept_remote", value="no")
- her_update_preference(dimension="relationship_goal", value="marriage")
```

## 注意事项

- 此 Skill 只是流程指南，Agent 需要根据实际情况灵活调整
- 用户可能跳过某些阶段，直接进入下一个
- 工具返回的是结构化数据，Agent 需要转换为自然的对话语言
