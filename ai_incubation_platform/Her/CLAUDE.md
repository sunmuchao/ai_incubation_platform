# Agent Native 架构核心原则

> **核心理念**：从"硬编码规则 + AI 调用"转向"AI 作为决策引擎 + 工具执行"

---

## 一、三层分离架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     System Prompt Layer                          │
│  所有决策逻辑通过 Prompt 表达                                     │
│  - 角色定义、决策规则、输出规范                                   │
│  - 不硬编码 if-else，用自然语言描述规则                           │
└─────────────────────────────────────────────────────────────────┘
                           ↓ 调用
┌─────────────────────────────────────────────────────────────────┐
│                     Tools Layer (Pure Execution)                 │
│  只做数据查询和执行，不含业务逻辑                                  │
│  - 返回原始数据（JSON），不做解读                                  │
│  - Agent 自己解读数据并决策下一步                                 │
└─────────────────────────────────────────────────────────────────┘
                           ↓ 依赖
┌─────────────────────────────────────────────────────────────────┐
│                     Data Layer                                   │
│  数据存储，无业务逻辑                                             │
└─────────────────────────────────────────────────────────────────┘
```

---

## 二、核心对比

| 维度 | 传统设计 | Agent Native 设计 |
|------|----------|-------------------|
| 规则表达 | 代码硬编码 | Prompt 表达 |
| 决策主体 | 规则引擎 | LLM Agent |
| 工具职责 | 包含业务逻辑 | 纯数据查询/执行 |
| 输出方式 | 预设模板 | AI 动态生成 |

---

## 三、执行循环

```
用户消息 → Agent 思考(LLM) → 选择工具 → 执行工具(纯数据) → 返回数据
                                                        ↓
                         Agent 再思考(LLM) ← 解读数据决定下一步
                                                        ↓
                         继续执行 或 输出回复 → 完成
```

---

## 四、开发规范

### 禁止

- ❌ Tool 中包含 if-else 业务判断
- ❌ 使用预设模板列表直接返回
- ❌ 代码硬编码"当 X 时输出 Y"

### 允许保留规则

- ✅ 商业限制（付费、次数）
- ✅ 安全边界（黑名单、封禁）
- ✅ 数据校验（必填、格式）
