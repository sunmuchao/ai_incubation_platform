📋 核心功能

📊

运行时指标

CPU、内存、P99、错误率等，可对接 Prometheus / OpenTelemetry。

👥

用户使用情况

路由热度、路径级延迟/错误、功能渗透率等，来自网关日志或埋点聚合。

🧠

综合分析

将指标与行为一起推理：热点慢路径、低采纳功能、错误与路由关联等。

📝

代码提案

输出可落地的变更草案（占位片段）；生产可接 LLM 生成 diff，经 CI 与人工评审后合并。

🔌 API 端点

  • POST/api/runtime/metrics指标快照
  • POST/api/runtime/usage用户使用聚合
  • POST/api/runtime/holistic-analyze指标+行为综合分析
  • POST/api/runtime/code-proposals代码变更草案
  • GET/api/runtime/recommendations最近建议

💡 使用场景

线上延迟与错误率波动排查
连接池、线程池等参数调优辅助
与全日志/追踪联动做根因
容量规划与压测结果对照
