# AI 员工出租平台 — 规划

## AI Native Agent 方向

将「员工选择 → 下单 → 状态推进 → 账单试算」封装为 Agent 可调工具集；租户侧交互以 **DeerFlow 2.0** 编排；训练数据版本、订单与 SLA 变更通过工具留痕，避免业务规则散落在 Prompt 中。

## 长期愿景

建设 **可训练、可计价、可审计** 的「数字员工」市场：租户按能力与 SLA 使用 AI 员工，支持训练数据版本化、租赁订单与评价闭环，而非一次性 Prompt 拼接。

## 孵化器 Agent 标准（摘要）

具备「LLM + 工具 + 多步编排」的路径统一使用 **DeerFlow 2.0**（经 `deerflow-integration/`）；业务动作封装为可调工具；敏感操作在工具层强校验与审计。全文见仓库根目录 `PLATFORM_AGENT_STANDARD.md`。

## AI Native 差异化战略

**❌ 我们不做**: 下一个 Upwork/Fiverr（拼雇主数量、拼品类）

**✅ 我们要做**: **可训练、可解释、可进化的 AI 员工市场**

### 竞品做法 vs AI Native 做法

| 竞品做法 | AI Native 做法 |
|---------|--------------|
| 人工筛选简历 | AI 匹配需求与员工能力 |
| 手动写工作描述 | AI 生成标准化需求文档 |
| 评价是静态分数 | AI 分析工作过程生成深度评价 |
| 交易完成就结束 | AI 追踪工作效果持续优化 |

### 我们的 AI 杀手级功能

| 功能 | 竞品状态 | 我们优势 |
|------|----------|---------|
| AI 需求分析师 | 无 | ✅ 把模糊需求转成标准化任务 |
| 智能匹配引擎 | 关键词匹配 | ✅ 能力图谱匹配 |
| 工作过程可观测 | 无 | ✅ AI 工作日志自动记录 |
| 训练效果评估 | 无 | ✅ A/B 测试哪个 AI 员工更好 |
| AI 员工进化 | 无 | ✅ 根据反馈自动微调 |

### 跨项目协同

- **代码理解** → 理解 AI 员工工作内容
- **运行态优化** → 监控 AI 员工性能
- **数据连接器** → 接入外部工作数据

## 差异化竞争策略

### AI Native 优势（建立壁垒）

| 优势 | 描述 | 实施路径 |
|------|------|---------|
| **训练即交付** | AI 员工训练完成 = 工作交付，无需人工验证 | 自动化工作日志 + 训练任务对接 |
| **可解释性** | Agent 决策过程全程可追溯 | 执行日志 + 决策树可视化 |
| **无限扩展** | AI 员工可同时服务多个客户 | 并发执行 + 资源隔离 |
| **持续进化** | 每次任务都是训练数据，越用越强 | 反馈闭环 + 增量训练 |
| **成本优势** | 边际成本趋近于零 | 规模效应 + 自动化 |

### 产品定位

**不建议**: 成为"AI 版 Upwork"（跟随策略）

**建议**: 成为"AI 员工经济基础设施"（引领策略）

差异化定位:
1. **不卖时间，卖结果**: AI 员工按交付付费，而非按小时付费
2. **不靠人力，靠算力**: 核心竞争力是训练数据和算法，而非人力规模
3. **不是外包，是产品**: AI 员工是标准化产品，可复制可迭代
